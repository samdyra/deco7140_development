Buttons

In order to create clear actions and feedback, we create buttons that support various use case. In this example, we have created 4 different buttons, with different labels that describe what the button used for. To improve the behavioral feedback, several states are also implemented, such as active, disabled, focused, and hovered.

In order to convey informations more clearly, we implement the principle Use of Color (WCAG 1.4.1). For example, call to actions colors will use the main colors, while disabled buttons will be grayed out.

Community Engagement Form

To minimize errors, and make the users fill the form in the community form, the form implement the WCAG 3.3.2 (Labels and Instructions) by adding labels on each of the form inputs. We also use input types and field constraints, for example in the email address, we have to format the email address based on an actual email address structure, if the field is not filled correctly, a red border will appear. We also indicate the required and optional field by * sign, and a feedback of red borders will appear if the required input is not filled.
